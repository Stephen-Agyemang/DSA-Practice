## 🛠️ The Problem-Solving Framework

To ensure my code is scalable, optimized, and built with systems-level intent, I tackle complex challenges out loud using a structured engineering methodology:

1.  **Understand:** Clarify constraints, identify edge cases, and establish explicit input/output parameters.
2.  **Match:** Connect the problem attributes to known data structures or algorithmic patterns.
3.  **Plan:** Draft the logic steps and verify boundaries before writing executable syntax.
4.  **Implement:** Translate the logical plan into clean, Pythonic, production-grade code.
5.  **Review:** Trace the code mentally with sample test cases to catch potential index out-of-bounds errors early.
6.  **Evaluate:** Analyze time and space complexity using Big-O notation to minimize auxiliary memory bottlenecks.
